Playwright là gì?

Một framework test tự động cho web do Microsoft phát triển. Viết code để máy tính tự click, gõ chữ, kiểm tra giao diện — thay vì làm bằng tay.

01.1 Vấn đề Playwright giải quyết

Hãy tưởng tượng bạn vừa sửa code login. Bạn phải mở Chrome, gõ email, gõ password, bấm nút, kiểm tra xem có vào được dashboard không. Mỗi lần deploy là làm lại. 10 trang web khác cũng vậy. Mệt.

Playwright giúp bạn viết một lần — máy chạy 100 lần. Mỗi commit, mỗi đêm, mỗi pull request đều có thể kiểm tra lại tự động.

01.2 Tại sao chọn Playwright?

⚡ tốc độ

Nhanh hơn Selenium

Giao tiếp với browser qua DevTools Protocol, không qua WebDriver chậm chạp.

🎯 ổn định

Auto-waiting

Tự đợi element xuất hiện, không cần viết sleep() rườm rà.

🌐 đa trình duyệt

Chromium, Firefox, WebKit

Một API duy nhất chạy được cả 3 browser engine chính.

🔧 đầy đủ

Tool xịn sẵn

Có codegen, trace viewer, test report — không cần lắp ghép thêm.

01.4 Khi nào KHÔNG dùng Playwright?

  • Test app mobile native (iOS/Android) → dùng Appium
  • Test desktop app (Electron là một ngoại lệ — Playwright có support)
  • Load testing / performance testing → dùng k6 hoặc JMeter
  • Unit test cho component nhỏ → dùng Jest hoặc Vitest

Cài đặt Playwright

Chỉ cần Node.js (≥ 18) và 3 phút. Lệnh init sẽ tự cài tất cả: dependencies, browsers, config, ví dụ test.

02.1 Cài bằng 1 lệnh

Mở terminal trong thư mục project, chạy:

bash
# Tạo project Playwright mới
npm init playwright@latest

Trình cài đặt sẽ hỏi vài câu:

  • TypeScript hay JavaScript? → Chọn TypeScript (gợi ý)
  • Thư mục test? → mặc định tests/
  • GitHub Actions workflow? → có (để biết CI hoạt động sao)
  • Cài browsers? → Có

02.2 Cấu trúc thư mục sinh ra

tree
my-project/
├── tests/
│   └── example.spec.ts        // test mẫu
├── tests-examples/
│   └── demo-todo-app.spec.ts  // test phức tạp hơn để tham khảo
├── playwright.config.ts       // cấu hình
├── package.json
└── .github/workflows/
    └── playwright.yml         // CI workflow

02.3 Chạy test đầu tiên

bash
# Chạy tất cả test
npx playwright test

# Chạy với UI mode (gợi ý cho người mới)
npx playwright test --ui

# Chỉ chạy 1 file
npx playwright test example.spec.ts

# Chạy trên 1 browser cụ thể
npx playwright test --project=chromium

# Xem report sau khi chạy
npx playwright show-report
Mẹo

Luôn dùng --ui khi mới học. UI mode cho thấy từng bước test chạy, có timeline, có DOM snapshot — học gấp 10 lần đọc tài liệu.

02.4 VS Code extension

Cài extension "Playwright Test for VSCode" (do Microsoft phát hành). Sau khi cài:

  • Có nút Run/Debug bên cạnh mỗi test
  • Record test bằng cách click vào browser
  • Pick locator bằng cách hover element
  • Hiển thị lỗi inline trong editor

Test đầu tiên

Cấu trúc một test Playwright và hiểu từng dòng làm gì. Bắt đầu từ ví dụ kinh điển: mở Google, gõ tìm kiếm.

03.1 Anatomy của 1 test

typescript
import { test, expect } from '@playwright/test';

test('có chữ Playwright trên title', async ({ page }) => {
  // 1. Đi tới trang web
  await page.goto('https://playwright.dev/');

  // 2. Kiểm tra title có chứa "Playwright"
  await expect(page).toHaveTitle(/Playwright/);
});

Giải thích từng phần:

  • import — nạp test và expect từ thư viện Playwright
  • test('tên', async ...) — định nghĩa 1 test case
  • { page } — Playwright tự cung cấp 1 trang web sạch cho mỗi test
  • await — mọi action với browser đều bất đồng bộ (async)
  • expect(...).toHave... — kiểm tra (assertion)

03.2 Test có tương tác

typescript
test('login thành công', async ({ page }) => {
  await page.goto('https://example.com/login');

  // Gõ vào ô email
  await page.getByLabel('Email').fill('user@test.com');

  // Gõ vào ô password
  await page.getByLabel('Password').fill('123456');

  // Click nút Submit
  await page.getByRole('button', { name: 'Đăng nhập' }).click();

  // Kiểm tra URL chuyển sang dashboard
  await expect(page).toHaveURL(/dashboard/);

  // Kiểm tra có chữ "Xin chào"
  await expect(page.getByText('Xin chào')).toBeVisible();
});

03.3 Codegen — record test tự động

Không cần gõ tay. Playwright có lệnh ghi lại hành động:

bash
npx playwright codegen https://example.com

Browser sẽ mở ra, bạn thao tác như bình thường (click, gõ, scroll). Playwright tự sinh code TypeScript tương ứng. Cực kỳ hữu ích để học cú pháp ban đầu.

Locators — Chỉ vào element

Locator là cách bạn nói với Playwright "tôi muốn thao tác với cái nút này, không phải cái khác." Đây là khái niệm quan trọng nhất. Học locator tốt = test ổn định.

04.1 Triết lý: ưu tiên user-facing

Playwright khuyên chọn locator giống cách người dùng thật nhận biết element — không phải CSS class hay ID trong code.

Lý do: developer đổi class name liên tục, nhưng "nút Đăng nhập" thì luôn là "nút Đăng nhập".

04.2 7 loại locator chính (theo thứ tự ưu tiên)

1. getByRole — theo ARIA role

typescript
page.getByRole('button', { name: 'Đăng nhập' })
page.getByRole('link', { name: 'Trang chủ' })
page.getByRole('checkbox', { name: 'Đồng ý điều khoản' })
page.getByRole('heading', { level: 1 })

2. getByLabel — input theo label

typescript
page.getByLabel('Email')
page.getByLabel('Password')
page.getByLabel('Tôi đồng ý')

3. getByPlaceholder — input chưa có label

typescript
page.getByPlaceholder('Nhập email của bạn')

4. getByText — theo chữ hiển thị

typescript
page.getByText('Welcome back')
page.getByText('Welcome', { exact: true })  // chính xác
page.getByText(/welcome/i)             // regex

5. getByTestId — qua data-testid attribute

typescript
// HTML: <button data-testid="submit-btn">Send</button>
page.getByTestId('submit-btn')

6. getByTitle — qua thuộc tính title

typescript
page.getByTitle('Issues count')

7. locator — CSS / XPath (cuối cùng mới dùng)

typescript
page.locator('.btn-primary')
page.locator('#main-form input[type="email"]')
page.locator('xpath=//button[contains(text(), "Save")]')

04.3 Kết hợp & lọc locator

typescript
// Có nhiều nút "Xóa" trên trang, chỉ lấy nút trong dòng có chữ "Apple"
page.getByRole('row', { name: 'Apple' })
    .getByRole('button', { name: 'Xóa' });

// Filter theo text
page.getByRole('listitem').filter({ hasText: 'Nike' });

// Filter theo locator con
page.getByRole('listitem').filter({
  has: page.getByRole('heading', { name: 'iPhone' })
});

// Lấy phần tử thứ N (index từ 0)
page.getByRole('button').nth(2);
page.getByRole('button').first();
page.getByRole('button').last();
Tránh xa

Đừng dùng page.locator('div > div > span.x-3 > button'). Chuỗi CSS dài là dấu hiệu locator sẽ vỡ ngay khi UI thay đổi nhẹ. Thay vào đó hỏi developer thêm data-testid.

Actions — Thao tác với element

Khi đã có locator, bạn làm gì với nó? Click, gõ chữ, chọn dropdown, upload file...

05.1 Click & các biến thể

typescript
await button.click();                          // click thường
await button.dblclick();                       // double-click
await button.click({ button: 'right' });        // chuột phải
await button.click({ modifiers: ['Shift'] }); // Shift+click
await button.click({ position: { x: 10, y: 20 } });
await button.click({ force: true });           // bỏ qua các kiểm tra
await button.hover();                          // rê chuột lên

05.2 Nhập liệu

typescript
// Điền ô input (xóa cũ, gõ mới)
await page.getByLabel('Email').fill('a@b.com');

// Gõ từng ký tự (giả lập gõ phím chậm)
await page.getByLabel('Note').pressSequentially('Hello', { delay: 100 });

// Xóa nội dung
await page.getByLabel('Email').clear();

// Bấm phím
await page.keyboard.press('Enter');
await page.keyboard.press('Control+A');
await page.getByRole('textbox').press('Tab');

05.3 Checkbox, radio, dropdown

typescript
// Checkbox / radio
await page.getByLabel('Đồng ý').check();
await page.getByLabel('Đồng ý').uncheck();
await page.getByLabel('Nam').check(); // radio button

// Dropdown <select>
await page.getByLabel('Quốc gia').selectOption('VN');          // theo value
await page.getByLabel('Quốc gia').selectOption({ label: 'Việt Nam' });
await page.getByLabel('Tags').selectOption(['red', 'green']);    // multi

05.4 Upload file

typescript
await page.getByLabel('Chọn file').setInputFiles('./avatar.png');

// Upload nhiều file
await page.getByLabel('Files').setInputFiles(['a.pdf', 'b.pdf']);

// Xóa file đã chọn
await page.getByLabel('Files').setInputFiles([]);

05.5 Drag & Drop, scroll, screenshot

typescript
// Drag & drop
await page.getByText('Item A').dragTo(page.getByTestId('dropzone'));

// Scroll đến element
await page.getByText('Footer').scrollIntoViewIfNeeded();

// Screenshot
await page.screenshot({ path: 'screen.png' });
await page.screenshot({ path: 'full.png', fullPage: true });
await button.screenshot({ path: 'just-button.png' });

Assertions — Kiểm tra kết quả

Assertion = "tôi mong đợi điều này phải đúng". Nếu sai, test fail. Playwright có 2 loại: web assertion (auto-retry) và assertion thường (so sánh giá trị tĩnh).

06.1 Web assertions là gì?

Web assertion là điểm cực mạnh của Playwright. Khi gọi expect(locator).toBeVisible(), Playwright sẽ:

  • Kiểm tra ngay lập tức
  • Nếu chưa đạt, đợi và kiểm tra lại liên tục
  • Đợi tối đa 5 giây (mặc định), nếu vẫn không đạt → test fail

Nghĩa là bạn không cần viết sleep(). Sau khi click submit, chỉ cần await expect(page.getByText('Thành công')).toBeVisible() — nó tự đợi.

Nhận biết web assertion

Có chữ await phía trước và đối số là locator hoặc page. Khác với assertion thường (so sánh giá trị: expect(5).toBe(5)).

06.2 Visibility & state — element hiển thị thế nào

toBeVisible / toBeHidden — có hiển thị không

Kiểm tra element có hiển thị trên trang không. toBeHidden ngược lại — element không có trong DOM hoặc bị ẩn (display:none, visibility:hidden, opacity:0).

typescript
await expect(page.getByText('Xin chào')).toBeVisible();
await expect(page.getByText('Loading...')).toBeHidden();

toBeAttached — element có trong DOM không

Element tồn tại trong DOM (kể cả ẩn). Hữu ích khi test element đã render nhưng chưa visible (vd: modal trước khi mở).

typescript
await expect(page.locator('#modal')).toBeAttached();
await expect(page.locator('#removed')).not.toBeAttached();

toBeEnabled / toBeDisabled — nút có bấm được không

Element có thuộc tính disabled hay không. Áp dụng cho button, input, select.

typescript
// Nút Submit phải bấm được
await expect(page.getByRole('button', { name: 'Submit' })).toBeEnabled();

// Khi chưa điền form, nút Submit phải disabled
await expect(page.getByRole('button', { name: 'Submit' })).toBeDisabled();

toBeEditable — ô input có cho gõ không

Element không bị disabled hoặc readonly.

typescript
await expect(page.getByLabel('Email')).toBeEditable();

toBeChecked — checkbox/radio đã tick chưa

Áp dụng cho checkbox, radio button, hoặc element có role="checkbox".

typescript
await expect(page.getByLabel('Đồng ý điều khoản')).toBeChecked();
await expect(page.getByLabel('Newsletter')).not.toBeChecked();

// Truyền checked: false thay vì .not.toBeChecked()
await expect(page.getByLabel('Newsletter')).toBeChecked({ checked: false });

toBeFocused — element đang được focus

Hữu ích khi test bàn phím (Tab giữa các input) hoặc autofocus.

typescript
await page.getByLabel('Email').press('Tab');
await expect(page.getByLabel('Password')).toBeFocused();

toBeEmpty — element rỗng

Element không có text con và không có element con nào. Hữu ích test list rỗng, container chưa load.

typescript
await expect(page.locator('#error-message')).toBeEmpty();

toBeInViewport — element trong vùng nhìn thấy

Element đang nằm trong viewport (vùng người dùng đang thấy), không phải ở phần đã cuộn ra ngoài.

typescript
await expect(page.getByText('Footer')).toBeInViewport();

// Tối thiểu 50% element phải nhìn thấy
await expect(loc).toBeInViewport({ ratio: 0.5 });

06.3 Text & value — nội dung element

toHaveText — text khớp chính xác

Text content của element phải khớp toàn bộ chuỗi (trim khoảng trắng đầu/cuối). Hỗ trợ string, regex, hoặc array (khi locator trỏ tới nhiều element).

typescript
// HTML: <h1>Welcome back, Hoa!</h1>
await expect(page.getByRole('heading')).toHaveText('Welcome back, Hoa!');

// Với regex
await expect(page.getByRole('heading')).toHaveText(/Welcome/);

// Với list — kiểm tra từng item
await expect(page.getByRole('listitem')).toHaveText(['Apple', 'Banana', 'Cherry']);

toContainText — text chứa chuỗi con

Element chứa chuỗi/regex (không cần khớp toàn bộ). Dùng khi text dài hoặc có phần thay đổi.

typescript
// HTML: <p>Đã đặt hàng thành công. Mã đơn: #A1B2C3</p>
await expect(page.getByRole('alert')).toContainText('thành công');
await expect(page.getByRole('alert')).toContainText(/Mã đơn: #\w+/);

toHaveValue — giá trị input

Áp dụng cho <input>, <textarea>, <select>.

typescript
await expect(page.getByLabel('Email')).toHaveValue('user@test.com');
await expect(page.getByLabel('Tuổi')).toHaveValue(/\d+/);

toHaveValues — multi-select đã chọn những gì

Dành riêng cho <select multiple> — kiểm tra danh sách option đang được chọn.

typescript
await page.getByLabel('Tags').selectOption(['red', 'green']);
await expect(page.getByLabel('Tags')).toHaveValues(['red', 'green']);

toHaveCount — số lượng element

Locator trỏ tới chính xác N element. Hữu ích cho list, grid, search results.

typescript
// Search trả về đúng 3 kết quả
await page.getByPlaceholder('Tìm').fill('iPhone');
await expect(page.getByRole('listitem')).toHaveCount(3);

// Không có kết quả nào
await expect(page.getByRole('listitem')).toHaveCount(0);

06.4 Attribute, class, CSS

toHaveAttribute — có attribute với giá trị

Element có HTML attribute. Hữu ích cho href, src, alt, data-*, aria-*, name, type...

typescript
// Link trỏ đúng URL
await expect(page.getByRole('link', { name: 'Về chúng tôi' }))
  .toHaveAttribute('href', '/about');

// Input có type đúng
await expect(page.getByLabel('Password'))
  .toHaveAttribute('type', 'password');

// Chỉ cần attribute tồn tại (không quan tâm giá trị)
await expect(page.getByRole('button')).toHaveAttribute('aria-pressed');

// Với regex
await expect(page.locator('img')).toHaveAttribute('src', /\.png$/);

toHaveClass — class chính xác

Khớp toàn bộ thuộc tính class. Thường dùng với regex để check 1 class trong nhiều class.

typescript
// HTML: <div class="btn btn-primary active">
// Khớp toàn bộ (ít dùng)
await expect(loc).toHaveClass('btn btn-primary active');

// Dùng regex để check 1 class (phổ biến)
await expect(loc).toHaveClass(/active/);

toContainClass — chứa class (mới, dễ dùng hơn)

Element chứa class chỉ định, dù còn class khác. Tốt hơn regex khi chỉ muốn check 1 class.

typescript
// HTML: <div class="btn btn-primary active">
await expect(loc).toContainClass('active');
await expect(loc).toContainClass('btn-primary active');  // nhiều class

toHaveCSS — giá trị CSS sau khi render

Kiểm tra CSS computed value (giá trị thật sau khi browser tính). Lưu ý màu sẽ trả về dạng rgb(), không phải hex.

typescript
// Nút lỗi có màu đỏ
await expect(page.getByRole('alert')).toHaveCSS('color', 'rgb(255, 0, 0)');

// Element ẩn
await expect(loc).toHaveCSS('display', 'none');

// Font size
await expect(loc).toHaveCSS('font-size', '16px');

toHaveId — element có id cụ thể

typescript
await expect(page.getByRole('main')).toHaveId('content');

toHaveJSProperty — JS property của element

Đôi khi cần kiểm tra property của DOM node (không phải HTML attribute) — ví dụ checked, value hiện tại sau khi user nhập, hoặc property tùy chỉnh.

typescript
await expect(page.getByLabel('Đồng ý')).toHaveJSProperty('checked', true);
await expect(page.locator('video')).toHaveJSProperty('paused', false);

toHaveRole — ARIA role của element

typescript
await expect(page.locator('#login-btn')).toHaveRole('button');

06.5 Page-level — kiểm tra cả trang

toHaveTitle — title trên tab browser

typescript
await expect(page).toHaveTitle('Trang chủ - Shop ABC');
await expect(page).toHaveTitle(/Shop ABC/);

toHaveURL — URL hiện tại của trang

Kiểm tra URL sau khi điều hướng (login → dashboard, click link, redirect...).

typescript
await page.getByRole('button', { name: 'Login' }).click();
await expect(page).toHaveURL(/\/dashboard/);

// URL chính xác
await expect(page).toHaveURL('https://shop.com/checkout?step=2');

toHaveScreenshot — visual regression

So sánh ảnh chụp màn hình với ảnh đã lưu trước đó. Lần đầu chạy sẽ tạo baseline, lần sau so sánh. Phát hiện lỗi UI mà không cần viết assertion cụ thể.

typescript
// Chụp & so sánh cả trang
await expect(page).toHaveScreenshot('home.png');

// Chỉ 1 element
await expect(page.getByRole('navigation')).toHaveScreenshot('nav.png');

// Cho phép sai lệch nhỏ
await expect(page).toHaveScreenshot('home.png', { maxDiffPixels: 100 });
Lần đầu

Lần đầu chạy toHaveScreenshot sẽ tạo file ảnh chuẩn và fail. Chạy lại sẽ pass. Để update khi UI thay đổi cố ý: npx playwright test --update-snapshots.

06.6 Phủ định với .not

Thêm .not trước assertion để đảo ngược ý nghĩa.

typescript
// Không hiển thị (sau khi đóng modal)
await expect(page.getByRole('dialog')).not.toBeVisible();

// Không chứa text lỗi
await expect(page.getByRole('alert')).not.toContainText('Lỗi');

// Chưa được tick
await expect(page.getByLabel('Newsletter')).not.toBeChecked();

// Không trỏ đến URL cũ
await expect(page).not.toHaveURL(/\/login/);

06.7 Tùy chỉnh timeout

Mặc định web assertion đợi tối đa 5 giây. Tăng/giảm tùy trường hợp:

typescript
// Element hiện chậm (vd: load PDF) — đợi 30s
await expect(page.getByText('PDF loaded')).toBeVisible({ timeout: 30000 });

// Muốn fail nhanh nếu element không xuất hiện ngay — 1s
await expect(page.getByText('Lỗi')).not.toBeVisible({ timeout: 1000 });

Đổi timeout mặc định cho toàn project trong playwright.config.ts:

typescript
export default defineConfig({
  expect: {
    timeout: 10000,  // 10s cho tất cả assertion
  },
});

06.8 Assertions thường (không có auto-retry)

Khi bạn có sẵn 1 giá trị cụ thể (số, chuỗi, object) — không phải locator — dùng assertion thường. Chúng so sánh 1 lần duy nhất, không đợi.

toBe / toEqual — so sánh giá trị

toBe so sánh chính xác (===), toEqual so sánh sâu cho object/array.

typescript
const count = await page.getByRole('listitem').count();
expect(count).toBe(5);

const user = await response.json();
expect(user).toEqual({ name: 'Hoa', age: 25 });

So sánh số: toBeGreaterThan, toBeLessThan, ...

typescript
expect(count).toBeGreaterThan(0);
expect(count).toBeGreaterThanOrEqual(5);
expect(count).toBeLessThan(100);

toMatch / toMatchObject — regex & object partial

typescript
const text = await page.getByRole('heading').textContent();
expect(text).toMatch(/Welcome/);

// Object có chứa các field này (không cần đủ tất cả field)
expect(user).toMatchObject({ name: 'Hoa' });

toBeTruthy / toBeFalsy / toBeNull

typescript
expect(user.id).toBeTruthy();        // không phải null/undefined/0/''
expect(error).toBeFalsy();
expect(result).toBeNull();
expect(result).toBeDefined();

toContain — array hoặc string chứa giá trị

typescript
expect(['red', 'green', 'blue']).toContain('green');
expect('Hello world').toContain('world');
Quan trọng

Web assertion (await expect(loc).toBeVisible()) ưu tiên hơn vì có auto-retry — tránh test flaky. Chỉ dùng assertion thường khi bạn đã có giá trị cụ thể (response API, text đã lấy ra, count đã đếm).

06.9 Soft assertions — không dừng test khi fail

Mặc định, assertion fail sẽ dừng test ngay. Đôi khi bạn muốn chạy tiếp để xem hết các lỗi → dùng expect.soft.

typescript
test('kiểm tra hết các field', async ({ page }) => {
  await page.goto('/profile');

  // Cả 3 assertion đều chạy dù có cái fail
  await expect.soft(page.getByLabel('Tên')).toHaveValue('Hoa');
  await expect.soft(page.getByLabel('Email')).toHaveValue('hoa@x.com');
  await expect.soft(page.getByLabel('Tuổi')).toHaveValue('25');

  // Test sẽ fail nếu có ít nhất 1 soft assertion fail
});

06.10 Custom error message

Thêm message để dễ debug khi test fail trong CI:

typescript
await expect(page.getByRole('button'), 'Nút Submit phải enable sau khi điền form')
  .toBeEnabled();

Auto-waiting — Tại sao test ổn định

Playwright tự động đợi element sẵn sàng trước khi thao tác. Hiểu cơ chế này = bớt 90% test flaky.

07.1 Playwright đợi cái gì?

Trước mỗi action như click(), Playwright kiểm tra element có:

  • Attached — tồn tại trong DOM
  • Visible — không bị display:none
  • Stable — không animation hoặc transition
  • Enabled — không bị disabled
  • Receives events — không bị element khác che

Nếu chưa đạt, Playwright tự đợi (mặc định tối đa 30 giây).

07.2 Khi nào cần đợi thủ công?

Hầu hết trường hợp không cần vì auto-wait đã lo. Nhưng có những lúc bạn cần chủ động đợi event xảy ra:

page.waitForURL — đợi điều hướng

Đợi URL khớp pattern. Thường dùng sau khi click → redirect.

typescript
await page.getByRole('button', { name: 'Login' }).click();
await page.waitForURL(/\/dashboard/);

// Hoặc URL chính xác
await page.waitForURL('https://shop.com/cart');
Lưu ý

Thường thì await expect(page).toHaveURL(...) đủ rồi (cũng auto-wait). Chỉ dùng waitForURL khi cần chắc chắn navigation xong trước khi làm tiếp.

page.waitForResponse — đợi API response

Đợi 1 network response cụ thể quay về. Hữu ích khi UI phụ thuộc vào data từ API.

typescript
// Click rồi đợi response /api/users về thành công
const responsePromise = page.waitForResponse(resp =>
  resp.url().includes('/api/users') && resp.status() === 200
);
await page.getByRole('button', { name: 'Load users' }).click();
const response = await responsePromise;

const users = await response.json();

page.waitForRequest — đợi request được gửi

Ngược lại với waitForResponse — đợi 1 request đi ra. Tốt để verify "click nút này có gọi API kia không".

typescript
const requestPromise = page.waitForRequest('**/api/track');
await page.getByRole('button', { name: 'Subscribe' }).click();
const request = await requestPromise;

expect(request.postDataJSON()).toMatchObject({ event: 'subscribe' });

page.waitForLoadState — đợi trang load xong

Đợi trạng thái cụ thể của trang. 3 mức:

  • 'load' — sự kiện load đã fire (mặc định của goto)
  • 'domcontentloaded' — DOM đã parse xong (không đợi ảnh, CSS)
  • 'networkidle' — không có request nào trong 500ms gần nhất
typescript
await page.goto('/');
await page.waitForLoadState('networkidle'); // đợi mọi request xong
Lưu ý

networkidle dễ làm test chậm và flaky nếu app có polling/websocket. Ưu tiên đợi element cụ thể (toBeVisible) thay vì network state.

page.waitForFunction — đợi điều kiện JS

Đợi đến khi 1 biểu thức JS trả về truthy. Dùng khi cần check state phức tạp không có locator tương ứng.

typescript
// Đợi biến global xuất hiện
await page.waitForFunction(() => window.dataLoaded === true);

// Đợi list có ít nhất 10 item
await page.waitForFunction(() =>
  document.querySelectorAll('.item').length >= 10
);

page.waitForSelector — đợi element xuất hiện (cũ)

Trước đây dùng để đợi element. Giờ nên dùng locator + assertion thay vì waitForSelector.

typescript
// ❌ Cũ
await page.waitForSelector('.item');

// ✅ Mới — gọn hơn, không bị deprecated
await expect(page.locator('.item')).toBeVisible();

07.3 ❌ Tuyệt đối tránh

typescript
// ❌ KHÔNG dùng hardcode sleep
await page.waitForTimeout(3000);

// ✅ Thay vào đó: đợi điều kiện cụ thể
await expect(page.getByText('Loaded')).toBeVisible();

waitForTimeout chỉ nên dùng tạm khi debug, không bao giờ trong test thật.

Fixtures & Hooks

Fixtures là cách chia sẻ setup giữa các test (login sẵn, dữ liệu test, browser custom). Hooks là code chạy trước/sau test.

08.1 Hooks cơ bản

typescript
import { test, expect } from '@playwright/test';

test.describe('Trang Sản phẩm', () => {
  test.beforeAll(async () => {
    // Chạy 1 lần trước tất cả test trong describe này
  });

  test.beforeEach(async ({ page }) => {
    // Chạy trước MỖI test
    await page.goto('/products');
  });

  test.afterEach(async ({ page }) => {
    // Chạy sau mỗi test (cleanup)
  });

  test.afterAll(async () => {
    // Chạy 1 lần sau tất cả
  });

  test('hiển thị danh sách', async ({ page }) => {
    await expect(page.getByRole('listitem')).toHaveCount(10);
  });
});

08.2 Built-in fixtures

Playwright cung cấp sẵn các fixture phổ biến:

FixtureLà gì?

08.3 Custom fixture — ví dụ login

Tránh viết code login lặp lại trong mọi test:

typescript
// fixtures.ts
import { test as base } from '@playwright/test';

type MyFixtures = {
  loggedInPage: import('@playwright/test').Page;
};

export const test = base.extend<MyFixtures>({
  loggedInPage: async ({ page }, use) => {
    await page.goto('/login');
    await page.getByLabel('Email').fill('test@a.com');
    await page.getByLabel('Password').fill('pass');
    await page.getByRole('button', { name: 'Login' }).click();
    await page.waitForURL(/dashboard/);
    await use(page);   // "trao" page đã login cho test
  },
});

export { expect } from '@playwright/test';
typescript
// dashboard.spec.ts
import { test, expect } from './fixtures';

test('xem profile', async ({ loggedInPage }) => {
  await loggedInPage.goto('/profile');
  await expect(loggedInPage.getByText('Hồ sơ')).toBeVisible();
});

08.4 Storage state — login 1 lần, dùng cho nhiều test

Page Object Model (POM)

Pattern phổ biến nhất trong test automation: gom locator và action của 1 trang vào 1 class. Dễ bảo trì, dễ đọc.

09.1 Vấn đề không có POM

Tưởng tượng 20 test đều có dòng page.getByLabel('Email').fill(...). Một ngày developer đổi thành "Email Address". Bạn phải sửa 20 chỗ.

09.2 POM mẫu

typescript
// pages/LoginPage.ts
import { Page, Locator } from '@playwright/test';

export class LoginPage {
  readonly page: Page;
  readonly emailInput: Locator;
  readonly passwordInput: Locator;
  readonly submitButton: Locator;
  readonly errorMessage: Locator;

  constructor(page: Page) {
    this.page = page;
    this.emailInput = page.getByLabel('Email');
    this.passwordInput = page.getByLabel('Password');
    this.submitButton = page.getByRole('button', { name: 'Login' });
    this.errorMessage = page.getByRole('alert');
  }

  async goto() {
    await this.page.goto('/login');
  }

  async login(email: string, password: string) {
    await this.emailInput.fill(email);
    await this.passwordInput.fill(password);
    await this.submitButton.click();
  }
}
typescript
// tests/login.spec.ts
import { test, expect } from '@playwright/test';
import { LoginPage } from '../pages/LoginPage';

test('login thành công', async ({ page }) => {
  const loginPage = new LoginPage(page);

  await loginPage.goto();
  await loginPage.login('a@b.com', '123456');

  await expect(page).toHaveURL(/dashboard/);
});

test('login sai password hiện lỗi', async ({ page }) => {
  const loginPage = new LoginPage(page);

  await loginPage.goto();
  await loginPage.login('a@b.com', 'sai');

  await expect(loginPage.errorMessage).toContainText('Sai mật khẩu');
});
Best practice

Locator nên là property của class (như trên), không phải method. Việc này cho phép dùng locator trong assertion thay vì chỉ trong action.

Network & API Testing

Playwright không chỉ test UI — nó cũng can thiệp được network: mock API, test endpoint trực tiếp, kiểm tra request đã gửi.

10.1 Mock API response với page.route

page.route cho phép bạn chặn request từ trang trước khi nó đi đến server thật, rồi quyết định: trả về data giả, sửa request, hoặc cho đi tiếp. Cực hữu ích để test các tình huống khó tái tạo (server lỗi, danh sách rỗng, data 1000 item...).

route.fulfill — trả về response giả

Dừng request và trả về data tự định nghĩa. Trang sẽ nhận data đó như thật.

typescript
test('hiển thị khi API trả 0 sản phẩm', async ({ page }) => {
  // Chặn request đến /api/products, trả về data rỗng
  await page.route('**/api/products', async route => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({ products: [] }),
    });
  });

  await page.goto('/shop');
  await expect(page.getByText('Không có sản phẩm')).toBeVisible();
});

Test lỗi server với route.fulfill status 500

Một trong những use-case quan trọng nhất — bình thường rất khó test:

typescript
test('hiển thị thông báo lỗi khi API 500', async ({ page }) => {
  await page.route('**/api/checkout', route =>
    route.fulfill({ status: 500, body: 'Server error' })
  );

  await page.goto('/cart');
  await page.getByRole('button', { name: 'Thanh toán' }).click();
  await expect(page.getByText('Có lỗi xảy ra, thử lại')).toBeVisible();
});

route.continue — sửa request rồi gửi đi thật

Cho request đi tiếp đến server nhưng có thể sửa header, body, URL.

typescript
// Thêm header xác thực vào mọi request API
await page.route('**/api/**', route => {
  route.continue({
    headers: {
      ...route.request().headers(),
      'Authorization': 'Bearer test-token',
    },
  });
});

route.abort — chặn không cho gọi

Hủy request hoàn toàn. Tốt để test khi ảnh không load, ads bị block, hay network fail.

typescript
// Chặn tất cả ảnh để test chạy nhanh hơn
await page.route('**/*.{png,jpg,jpeg,gif,webp}', route => route.abort());

// Test khi 3rd party script bị block
await page.route('**/google-analytics.com/**', route => route.abort());

10.2 Pure API test (không cần browser)

Playwright có thể gọi API trực tiếp như Postman, không cần mở browser. Tốt cho:

  • Test endpoint API riêng biệt
  • Setup data trước test UI (vd: tạo user qua API → login UI)
  • Verify backend song song với UI

request.post / request.get / ...

typescript
test('API: tạo user', async ({ request }) => {
  const response = await request.post('/api/users', {
    data: { name: 'Hoa', email: 'hoa@x.com' },
    headers: { 'Authorization': 'Bearer token123' },
  });

  expect(response.status()).toBe(201);
  expect(response.ok()).toBeTruthy();

  const body = await response.json();
  expect(body.name).toBe('Hoa');
});

Các method khác

typescript
await request.get('/api/users/1');
await request.put('/api/users/1', { data: { name: 'New name' } });
await request.patch('/api/users/1', { data: { age: 30 } });
await request.delete('/api/users/1');

// Query params
await request.get('/api/users', {
  params: { page: 2, limit: 20 },
});

Setup data trước test UI

typescript
test('user đã có order xem được lịch sử', async ({ page, request }) => {
  // Setup: tạo order qua API (nhanh hơn click qua UI)
  await request.post('/api/orders', {
    data: { userId: 1, items: ['iPhone'] },
  });

  // Test: vào trang lịch sử, kiểm tra hiển thị
  await page.goto('/orders');
  await expect(page.getByText('iPhone')).toBeVisible();
});

10.3 Kiểm tra request được gửi đi

Đôi khi bạn cần verify "click nút này có thật sự gọi API kia với đúng payload không". Dùng waitForRequest kết hợp với action.

Pattern: Promise.all + waitForRequest

Cách an toàn: setup listener trước khi action xảy ra (vì request có thể nhanh hơn code Promise tiếp theo).

typescript
test('click submit gửi đúng payload', async ({ page }) => {
  await page.goto('/contact');
  await page.getByLabel('Tên').fill('Hoa');

  // Đợi request gửi đi đồng thời click submit
  const [request] = await Promise.all([
    page.waitForRequest('**/api/contact'),
    page.getByRole('button', { name: 'Gửi' }).click(),
  ]);

  // Verify payload
  expect(request.method()).toBe('POST');
  expect(request.postDataJSON()).toMatchObject({
    name: 'Hoa',
  });
});

Truy cập thông tin request

typescript
request.url();           // URL đầy đủ
request.method();        // GET, POST, PUT, ...
request.headers();       // object { header: value }
request.postData();      // body dạng string
request.postDataJSON();  // body parse JSON

Khi test fail, bạn cần biết tại sao. Playwright có UI Mode, Trace Viewer, Inspector — 3 vũ khí mạnh nhất.

bash

bash

CI/CD — Chạy tự động

Mục tiêu cuối: mỗi pull request đều tự động chạy hết test, fail thì không cho merge.

12.1 GitHub Actions (ví dụ đầy đủ)

yaml
# .github/workflows/playwright.yml
name: Playwright Tests
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  test:
    timeout-minutes: 60
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 'lts/*'
      - name: Install dependencies
        run: npm ci
      - name: Install Playwright Browsers
        run: npx playwright install --with-deps
      - name: Run Playwright tests
        run: npx playwright test
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 30

12.2 Tối ưu CI

  • Chạy song song — Playwright tự động dùng nhiều worker
  • Retry on flaky — set retries: 2 trong config (chỉ áp dụng cho CI)
  • Shard — chia test ra nhiều machine: --shard=1/3, --shard=2/3...
  • Cache browsers — không tải lại Chromium mỗi lần
typescript
// playwright.config.ts
export default defineConfig({
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: process.env.CI ? 'github' : 'list',
});
